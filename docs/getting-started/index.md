# Getting started

Three docs take you from nothing to a running server:

1. **[Installation](installation.md)** — install Firecracker, build the
   binaries, create the guest rootfs.
2. **[Quickstart](quickstart.md)** — run your first task and understand what
   just happened.
3. **[Configuration](configuration.md)** — tune paths, bind addresses, auth,
   egress, rate limits.

If you just want to tick the tyres, start with the quickstart. If you're
deploying properly, read all three in order.
